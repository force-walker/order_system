Any order changes after cutoff must require a reason code and be audit-logged.

After such changes, procurement list data must be recalculated and regenerated per delivery note / business day.

Previous generated versions must be retained with version number or regeneration timestamp.

If regenerated required quantity differs from allocated quantity, the system must flag under-allocation or over-allocation for user review.
